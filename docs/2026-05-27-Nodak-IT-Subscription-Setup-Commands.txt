================================================================================
Nodak IT subscription — one-time admin setup for prod Claims Analyzer VM
================================================================================

Context
-------
Required to provision the prod ImageRight VPN proxy VM inside the
ClaimsAnalyzerTool resource group. Four ops at subscription scope are
above sb@spej.ai's RG-Contributor access:

  1. Register Microsoft.Compute
  2. Register Microsoft.Network
  3. Create + assign a custom "Resource Provider Registrar" role so
     sb@spej.ai can register future providers without admin help.
  4. Grant sb@spej.ai "Key Vault Secrets Officer" on kv-nodak-prod-88db
     (vault already exists in the RG).

Subscription: Nodak IT
Subscription ID: 7d4915a8-1ee4-4c01-8930-e0d5563d97fe
Tenant: nodakins.com
Resource group: ClaimsAnalyzerTool


Commands
--------

  az login
  az account set --subscription 7d4915a8-1ee4-4c01-8930-e0d5563d97fe
  SUB="/subscriptions/7d4915a8-1ee4-4c01-8930-e0d5563d97fe"

  # 1, 2 -- register providers
  az provider register --namespace Microsoft.Compute
  az provider register --namespace Microsoft.Network

  # 3 -- custom role + assignment
  cat > /tmp/rp-registrar.json <<'EOF'
  {
    "Name": "Resource Provider Registrar",
    "Description": "Register Azure resource providers; read provider state.",
    "IsCustom": true,
    "Actions": ["Microsoft.Resources/subscriptions/providers/read", "*/register/action"],
    "NotActions": [],
    "AssignableScopes": ["/subscriptions/7d4915a8-1ee4-4c01-8930-e0d5563d97fe"]
  }
  EOF
  az role definition create --role-definition /tmp/rp-registrar.json
  az role assignment create --assignee sb@spej.ai \
    --role "Resource Provider Registrar" --scope "$SUB"

  # 4 -- KV data-plane access
  az role assignment create --assignee sb@spej.ai \
    --role "Key Vault Secrets Officer" \
    --scope "$SUB/resourceGroups/ClaimsAnalyzerTool/providers/Microsoft.KeyVault/vaults/kv-nodak-prod-88db"


Verifying
---------

Provider registration is async; allow 2-5 minutes after the register
calls. Then run:

  az provider show --namespace Microsoft.Compute --query registrationState -o tsv
  az provider show --namespace Microsoft.Network --query registrationState -o tsv

Both should return: Registered


Portal alternative
------------------

  Steps 1, 2: Subscriptions > Nodak IT > Resource providers > Register on both
  Step 3:     IAM > Add custom role (JSON tab, paste the Actions list)
              then IAM > Add role assignment
  Step 4:     Open the kv-nodak-prod-88db vault > IAM > Add role assignment
